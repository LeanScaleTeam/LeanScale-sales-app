## Milestone 1: Quote to Cash - 1. Discovery & Configuration
**Tag:** `quote-to-cash`
**Description:** Discovery, vendor selection, CPQ configuration, and CLM integration setup
**Hours:** 75h

### Task List: (Quote to Cash) 1. Discovery & Vendor Selection
**Contains:** Parts 1-2

| Task | Est | Description |
|------|-----|-------------|
| 1. Map the Existing Q2C Process Flow | 4h | Document the complete current-state workflow from quote creation through cash collection, identifying all systems, handoff points, and manual processes. | 2. Quantify Revenue Leakage and Inefficiencies | 4h | Measure the financial impact of current Q2C gaps including billing errors, payment delays, and manual effort costs. End state: Business case document with quantified losses and improvement targets.<br /><br />• Pull billing error rate data from finance (incorrect invoices, credit memos, disputes)<br />• Calculate involuntary churn from payment failures (industry average is 0.8% of MRR)<br />• Measure time spent on manual quote creation, approval chasing, and invoice generation<br />• Identify revenue recognition issues or audit findings from past periods<br />• Document compliance gaps (ASC 606/IFRS 15 manual workarounds)<br />• Quantify the gap (e.g., "15% of invoices require manual correction, costing 20 hours/week") |
| 3. Define Requirements and Success Metrics | 3.5h | Gather requirements from all stakeholder groups and establish measurable success criteria. End state: Requirements document with prioritized features and KPIs.<br /><br />• Conduct requirements workshops with Sales (quoting needs), Finance (billing/rev rec needs), Legal (contract needs)<br />• Document must-have vs. nice-to-have capabilities<br />• Define approval workflow requirements (discount thresholds, deal desk involvement)<br />• Establish success metrics: cycle time reduction target, error rate target, automation percentage<br />• Get executive sign-off on requirements and success criteria |
| 4. Build Evaluation Criteria and Shortlist | 3.5h | Create structured evaluation framework and identify candidate Q2C platforms based on requirements. End state: Shortlist of 3-4 vendors with evaluation scorecard criteria.<br /><br />• Define evaluation criteria: CPQ capabilities, CLM features, billing automation, revenue recognition, integration depth<br />• Weight criteria based on client priorities (e.g., compliance vs. speed vs. cost)<br />• Research candidate platforms (DealHub, Subskribe, Maxio, Salesforce Revenue Cloud, Nue.io, Conga)<br />• Note: Salesforce legacy CPQ entering End of Sale March 2025; consider Revenue Cloud for SF shops<br />• Create shortlist of 3-4 vendors that match tech stack (CRM, ERP compatibility) |
| 5. Conduct Vendor Demos and POC | 6h | Run structured vendor demonstrations and proof-of-concept tests on core workflows. End state: Completed vendor scorecards with demo notes and POC results.<br /><br />• Schedule vendor demos with standardized agenda covering all key use cases<br />• Prepare demo scenarios: standard quote, complex deal with discounts, contract amendment, invoice generation<br />• Evaluate ease of use for sales reps (adoption is critical)<br />• Test integration approach with existing CRM (Salesforce/HubSpot) and ERP (NetSuite/SAP)<br />• Score each vendor against evaluation criteria<br />• Conduct POC with top 1-2 vendors on actual client data if possible |
| 6. Finalize Selection and Procurement | 3h | Make final vendor recommendation, negotiate contracts, and complete procurement. End state: Signed vendor agreement with implementation timeline.<br /><br />• Present recommendation to stakeholders with pros/cons analysis<br />• Negotiate licensing terms (per-user vs. transaction-based, annual vs. multi-year)<br />• Clarify implementation support included vs. additional cost<br />• Review data security and compliance certifications (SOC 2, GDPR)<br />• Finalize contract and initiate procurement process<br />• Establish implementation kickoff date |

### Task List: (Quote to Cash) 2. CPQ & CLM Configuration
**Contains:** Parts 3-4

| Task | Est | Description |
|------|-----|-------------|

End state: Complete product catalog with pricing logic that mirrors current rate card and business rules.<br /><br />• Import or build product catalog with all SKUs, bundles, and service offerings<br />• Configure pricing tiers (volume-based, customer segment-based)<br />• Set up discount approval thresholds (e.g., &lt;10% auto-approve, 10-20% manager, >20% deal desk)<br />• Build pricing rules for common scenarios (multi-year discounts, payment term adjustments)<br />• Ensure SKU naming conventions match billing system to prevent reconciliation issues<br />• Test pricing calculations against known good quotes |
| 8. Build Quote Templates and Document Generation | 6h | Create quote document templates and configure automated document generation. End state: Professional quote templates that auto-populate with deal data and generate PDFs.<br /><br />• Design quote template with branding, terms summary, and pricing table<br />• Configure dynamic fields that pull from CRM opportunity and CPQ configuration<br />• Set up multiple templates for different scenarios (new business, renewal, expansion)<br />• Build approval stamp/signature block for internal approvals<br />• Test document generation with sample data<br />• Get Sales and Legal sign-off on template format |
| 9. Configure Approval Workflows | 6h | Build multi-stage approval workflows based on deal characteristics. End state: Automated approval routing that enforces discount policies and deal desk requirements.<br /><br />• Map current approval matrix (who approves what based on discount, deal size, terms)<br />• Configure approval workflow stages in CPQ<br />• Set up notifications and escalation paths for stalled approvals<br />• Build deal desk queue for complex deals requiring manual review<br />• Configure mobile/email approval capabilities for managers<br />• Test approval routing with scenarios at each threshold level |
| 10. Configure Contract Templates and Clause Library | 8h | Set up contract templates and build a clause library for standard and negotiated terms. End state: Master service agreement and order form templates with modular clause options.<br /><br />• Import or create MSA, Order Form, and Amendment templates<br />• Build clause library with pre-approved language options (payment terms, liability caps, SLAs)<br />• Configure fallback clauses for negotiation scenarios<br />• Set up contract metadata fields (effective date, term length, renewal terms, auto-renewal)<br />• Integrate Legal playbook guidance for when to escalate non-standard terms<br />• Test template generation from CPQ quote data |
| 11. Build Contract-to-Quote Data Flow | 10h | Establish bidirectional data sync between CPQ and CLM so quotes automatically generate contracts and contract changes update deal records. End state: Seamless quote-to-contract flow without manual data re-entry.<br /><br />• Map CPQ fields to CLM contract fields (products, pricing, customer info)<br />• Configure automatic contract generation when quote reaches "approved" status<br />• Build sync for contract status back to CRM opportunity (sent, negotiating, signed)<br />• Set up version control for contract amendments during negotiation<br />• Configure signature routing through e-signature tool (DocuSign, PandaDoc, Ironclad)<br />• Test end-to-end flow: quote approval triggers contract, signature triggers deal close |
| 12. Configure Amendment and Renewal Workflows | 13h | Build workflows for mid-contract amendments and renewal scenarios. End state: Amendment and renewal processes that keep CPQ, CLM, and billing in sync.<br /><br />• Configure amendment workflow for add-ons, upgrades, and downgrades<br />• Build proration logic for mid-term changes<br />• Set up renewal notification triggers (90/60/30 days before expiration)<br />• Configure auto-renewal handling vs. manual renewal process<br />• Establish version tracking so amendments reference original contract<br />• Test amendment scenario: customer adds seats mid-contract, verify proration calculates correctly |

## Milestone 2: Quote to Cash - 2. Billing & Revenue Recognition
**Tag:** `quote-to-cash`
**Description:** Billing system integration, invoicing, revenue recognition, and compliance setup
**Hours:** 45h

### Task List: (Quote to Cash) 3. Billing & Revenue Recognition
**Contains:** Parts 5-6

| Task | Est | Description |
|------|-----|-------------|
| 13. Map Contract Data to Billing Records | 8h | Configure the integration between CLM/CPQ and billing system so signed contracts automatically create billing records. End state: Signed contracts trigger invoice schedules without manual billing setup.<br /><br />• Map contract fields to billing system fields (customer, products, pricing, dates, payment terms)<br />• Configure billing record creation trigger (contract signed, start date, etc.)<br />• Set up invoice schedule generation (monthly, quarterly, annual, milestone-based)<br />• Handle proration logic for mid-period starts<br />• Build SKU mapping table to resolve any naming differences between CPQ and billing<br />• Test: signed contract generates correct invoice schedule automatically |
| 14. Configure Invoice Generation and Delivery | 7h | Set up automated invoice generation, formatting, and delivery to customers. End state: Invoices auto-generate and deliver on schedule with correct line items and amounts.<br /><br />• Configure invoice template with branding and required fields (PO number, billing contact)<br />• Set up invoice generation schedule (day of month, X days before due date)<br />• Configure delivery methods (email, customer portal, EDI for enterprise customers)<br />• Build dunning sequence for overdue invoices (reminder at 7/14/30 days)<br />• Set up credit memo and refund workflows for billing corrections<br />• Test invoice generation and verify amounts match contract terms |
| 15. Connect Payment Collection and Cash Application | 8h | Integrate payment gateways and configure cash application to close the loop from invoice to collected revenue. End state: Payments auto-apply to invoices with minimal manual reconciliation.<br /><br />• Integrate payment gateway(s) (Stripe, ACH, wire, credit card)<br />• Configure automatic payment for customers on recurring billing<br />• Set up payment matching rules for manual payments (check, wire)<br />• Build failed payment retry logic and dunning escalation<br />• Configure cash application to match payments to open invoices<br />• Monitor involuntary churn metrics and payment failure rates |
| 16. Configure Revenue Recognition Rules | 8h | Set up ASC 606/IFRS 15 compliant revenue recognition in the billing or ERP system. End state: Revenue schedules auto-generate based on contract terms and performance obligations.<br /><br />• Document client's revenue recognition policy (point-in-time vs. over time, performance obligations)<br />• Configure rev rec rules in billing/ERP system for each product type<br />• Set up deferred revenue handling for prepaid contracts<br />• Build revenue schedule generation from contract terms (pro-rata, milestone, usage-based)<br />• Configure handling for multi-element arrangements (software + services bundles)<br />• Validate rev rec treatment with Finance/Accounting team |
| 17. Build Audit Trail and Compliance Reporting | 7h | Establish audit-ready documentation and compliance reporting for revenue transactions. End state: Complete audit trail from quote through recognized revenue with compliance reports.<br /><br />• Configure audit logging for all quote, contract, invoice, and payment changes<br />• Build compliance reports for ASC 606 disclosure requirements<br />• Set up data retention policies for quotes, contracts, and invoices (7+ years typical)<br />• Document revenue recognition methodology for auditors<br />• Create reconciliation reports: bookings vs. billings vs. revenue<br />• Test audit trail by tracing sample deal from quote through rev rec |
| 18. Set Up Financial System Integration | 7h | Configure integration between billing system and general ledger/ERP for financial close. End state: Billing data flows to GL automatically with proper account mapping.<br /><br />• Map billing line items to GL account codes (revenue, deferred revenue, AR)<br />• Configure journal entry generation frequency (daily, weekly, real-time)<br />• Build reconciliation reports between billing system and GL<br />• Set up month-end close automation for revenue reports<br />• Test month-end close process with Finance team<br />• Document close calendar and integration touchpoints |
