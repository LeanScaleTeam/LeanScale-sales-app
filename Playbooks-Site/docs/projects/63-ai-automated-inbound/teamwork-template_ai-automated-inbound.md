## Milestone 1: AI Automated Inbound - 1. Discovery, Design & Integration
**Tag:** `ai-inbound`

### Task List: (AI Automated Inbound) 1. Discovery & Playbook Design
**Contains:** Parts 1-2

| Task | Est | Description |
|------|-----|-------------|
| 1. Conduct Stakeholder Discovery Session | 2.5h | Meet with Marketing, Sales, and RevOps leadership to align on inbound engagement goals and define success metrics. End state: Documented goals for MQL-to-SQL conversion, speed-to-lead targets, and meeting booking rates.<br /><br />• Schedule 60-90 minute discovery session with Marketing, Sales Ops, and Sales leadership<br />• Define primary inbound engagement goals (e.g., 2X meetings booked, &lt;5 min speed-to-lead)<br />• Document current MQL-to-SQL conversion rate as baseline<br />• Identify key buyer segments and their typical on-site behaviors<br />• Clarify budget and timeline constraints for tool selection<br />• Capture stakeholder expectations for AI vs. human handoff scenarios |
| 2. Audit Current Inbound Workflows | 2.5h | Assess existing inbound processes, website visitor data, and qualification criteria to identify gaps and opportunities. End state: Gap analysis showing current state vs. desired state with specific improvement areas.<br /><br />• Map current lead capture mechanisms (forms, demo requests, contact pages)<br />• Pull speed-to-lead metrics from CRM (time from form submission to first response)<br />• Document existing qualification criteria (BANT, MEDDIC, or custom framework)<br />• Identify high-traffic pages where inbound intent signals are strongest<br />• Review current routing rules and SLA compliance rates<br />• Quantify missed opportunities (e.g., "40% of demo requests never get follow-up within 24 hours") |
| 3. Evaluate and Select AI Inbound Tool | 3h | Compare AI inbound platform options against client's tech stack, requirements, and budget. End state: Tool selected (e.g., Qualified.com, Drift) with procurement approved.<br /><br />• Document client's current tech stack (CRM, MAP, website platform, existing chat tools)<br />• Evaluate options: Qualified.com, Drift, Intercom, HubSpot Conversations<br />• Compare on: CRM compatibility, AI capabilities, meeting booking, analytics depth, cost per seat<br />• Assess integration complexity with existing systems<br />• Present recommendation with ROI projections to stakeholders<br />• Get budget approval and initiate procurement/contract process |
| 4. Map Buyer Segments and On-Site Behaviors | 3h | Define key buyer personas and the on-site behaviors that indicate buying intent for each segment. End state: Documented buyer segment map with intent signals and qualification criteria per segment.<br /><br />• Identify 3-5 primary buyer segments (e.g., enterprise vs. SMB, by industry, by role)<br />• Map high-intent behaviors per segment (pricing page visits, demo page, security pages)<br />• Define qualification criteria for each segment using existing framework<br />• Document account signals (company size, industry, tech stack) that indicate fit<br />• Create segment-specific routing rules (e.g., enterprise → AE, SMB → SDR)<br />• Validate segment definitions with Sales leadership |
| 5. Design Conversational Playbooks | 4h | Create AI chat playbooks with qualification questions, value prompts, and objection handling for each buyer segment. End state: Complete conversational flow documents ready for tool configuration.<br /><br />• Draft 3-5 conversational playbooks aligned to buyer segments and intent levels<br />• Define opening messages based on page context (pricing vs. blog vs. product pages)<br />• Create qualification question sequences (3-5 questions per flow)<br />• Write value prompts and benefit statements for key objections<br />• Define escalation triggers for human handoff (complex questions, high-value accounts)<br />• Include meeting booking CTAs at optimal points in conversation flow<br />• Document fallback responses for out-of-scope questions |
| 6. Define Routing Rules and Escalation Logic | 3h | Establish routing rules for hot leads, existing customers, target accounts, and non-fit visitors. End state: Complete routing matrix with owner assignments and SLA targets.<br /><br />• Create routing matrix: segment × intent level × owner/queue<br />• Define rules for existing customers (route to CSM or dedicated queue)<br />• Set up ABM-specific routing for target accounts (priority queue, specific AE)<br />• Establish non-target visitor handling (resource offer, nurture path, graceful exit)<br />• Define SLA targets for each queue (e.g., hot leads &lt;2 min, standard &lt;5 min)<br />• Document after-hours routing logic (meeting booking only vs. queue for morning) |

### Task List: (AI Automated Inbound) 2. Integration & Configuration
**Contains:** Parts 3-4

| Task | Est | Description |
|------|-----|-------------|
| 7. Connect AI Platform to CRM | 3h | Establish bidirectional connection between Qualified.com (or selected tool) and CRM with proper API permissions. End state: AI platform connected to CRM with leads/contacts syncing correctly.<br /><br />• Configure OAuth connection to Salesforce or HubSpot<br />• Grant required API permissions (read/write leads, contacts, accounts, activities)<br />• Map AI platform fields to CRM standard and custom fields<br />• Set up lead/contact creation rules (create new vs. update existing)<br />• Configure account matching logic for known visitors<br />• Test bidirectional sync with sample records<br />• Document integration settings for client handoff |
| 8. Set Up Data Enrichment Integration | 3h | Connect data enrichment tools to enable real-time visitor identification and account scoring. End state: Enrichment flowing into AI platform for personalized, data-driven conversations.<br /><br />• Connect enrichment tools (Clearbit, ZoomInfo, 6sense, or native enrichment)<br />• Configure reverse IP lookup for company identification<br />• Set up contact enrichment for known visitors (title, seniority, department)<br />• Define enrichment triggers (on page load vs. on engagement)<br />• Map enriched data to AI platform variables for personalization<br />• Test enrichment accuracy with sample visitors |
| 9. Integrate Marketing Automation Platform | 3h | Connect AI platform to MAP for lead scoring sync and nurture workflow triggers. End state: Leads scored correctly with automated nurture enrollment based on AI interactions.<br /><br />• Connect to HubSpot, Marketo, or Pardot via native integration or API<br />• Configure lead score sync from MAP to AI platform (for prioritization)<br />• Set up activity logging from AI platform to MAP (conversation events)<br />• Define nurture enrollment triggers based on AI qualification outcomes<br />• Configure lead source/campaign tracking for attribution<br />• Test end-to-end flow: visitor → AI chat → CRM lead → MAP nurture |
| 10. Configure AI Chatbot Foundation | 3.5h | Set up core AI chatbot configuration including appearance, behavior settings, and base AI training. End state: AI chatbot deployed on website with foundational settings in place.<br /><br />• Configure chatbot appearance (colors, logo, positioning) to match brand<br />• Set up bot personality and tone of voice aligned with company brand guidelines<br />• Configure AI training with company-specific context (product info, pricing tiers, competitors)<br />• Set up knowledge base connections for accurate responses<br />• Define bot behavior by page type (aggressive on pricing, passive on blog)<br />• Configure mobile vs. desktop experience differences<br />• Set initial AI confidence thresholds for escalation |
| 11. Implement Conversational Playbooks | 4h | Build out the designed conversational flows in the AI platform with all qualification paths and routing logic. End state: All playbooks configured and ready for testing.<br /><br />• Build each conversational playbook in the platform<br />• Configure conditional branching based on visitor responses<br />• Set up qualification data capture (company size, use case, timeline, budget)<br />• Implement routing logic per playbook (trigger assignments, queue selections)<br />• Configure meeting booking integration (calendar connections, availability rules)<br />• Set up real-time alerts for high-priority visitors (Slack, email, SMS)<br />• Add personalization tokens using enrichment data |
| 12. Configure Meeting Booking Experience | 3.5h | Set up real-time meeting booking with calendar integrations and availability rules. End state: Visitors can book meetings directly from AI conversations with correct rep calendars.<br /><br />• Connect rep calendars (Google Calendar or Outlook) to AI platform<br />• Configure availability windows and buffer times per rep<br />• Set up round-robin or weighted distribution logic<br />• Define meeting types (15 min intro, 30 min demo, 60 min deep dive)<br />• Configure confirmation emails and calendar invites<br />• Set up rescheduling and cancellation workflows<br />• Test booking flow end-to-end with multiple rep calendars |
| 13. QA and Test All AI Experiences | 4h | Conduct comprehensive testing of all conversational flows, routing, and integrations before launch. End state: All flows tested, issues resolved, and system ready for pilot launch.<br /><br />• Test each playbook flow manually (happy path and edge cases)<br />• Verify routing delivers to correct owner/queue for each scenario<br />• Test CRM record creation and field mapping accuracy<br />• Verify enrichment data appearing correctly in conversations<br />• Test meeting booking across all rep calendars and time zones<br />• Validate mobile experience functionality<br />• Test after-hours behavior and fallback scenarios<br />• Document any issues and resolve before pilot |
