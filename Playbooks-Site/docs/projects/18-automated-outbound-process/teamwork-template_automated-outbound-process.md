## Milestone 1: Automated Outbound Process - 1. Strategy, Tools & Data Infrastructure
**Tag:** `automated-outbound`
**Description:** Assess current outbound operations, select and procure tools, and build target lists with data infrastructure
**Hours:** 52h

### Task List: (Automated Outbound) 1. Strategy & Tool Selection
**Contains:** Parts 1-2

| Task | Est | Description |
|------|-----|-------------|
| 1. Audit Current Outbound Operations | 4h | Evaluate the existing outbound process to understand what's working, what's broken, and where automation can have the highest impact. End state: Gap analysis document showing current state vs. desired state.<br /><br />• Interview SDR team leads to understand current workflow and pain points<br />• Review existing sequence performance data (open rates, reply rates, conversion rates)<br />• Audit current tech stack usage and identify underutilized tools<br />• Document manual processes that consume SDR time (list building, research, data entry)<br />• Quantify the gap (e.g., "SDRs spend 4 hours/day on non-selling activities")<br />• Map current handoff points between marketing, SDR, and AE teams |
| 2. Define Outbound Strategy and Goals | 3h | Clarify the strategic objectives, target segments, and success metrics for the automated outbound program. End state: Documented outbound strategy with clear KPIs and targets.<br /><br />• Define primary outbound goals (meetings booked, pipeline generated, revenue targets)<br />• Identify target segments and prioritization criteria (industry, company size, use case)<br />• Set channel mix strategy (email-heavy vs. multi-channel with phone and social)<br />• Establish baseline metrics and improvement targets (e.g., "increase meetings booked by 40%")<br />• Align outbound strategy with marketing campaigns and inbound handoff processes<br />• Document decision criteria for tool selection phase |
| 3. Evaluate Data Enrichment Tools | 4h | Compare data enrichment providers to determine the best fit for the client's ICP and data needs. End state: Data enrichment tool selected with procurement initiated.<br /><br />• Define data requirements (firmographics, technographics, contact info, intent signals)<br />• Evaluate options: ZoomInfo, Apollo, Clearbit, Lusha, Cognism based on coverage and accuracy<br />• Run test enrichment on sample of 100 target accounts to compare data quality<br />• Compare pricing models (per-record vs. subscription vs. usage-based)<br />• Assess integration capabilities with existing CRM and sales engagement platform<br />• Present recommendation to client with ROI analysis and get budget approval |
| 4. Evaluate and Select Sales Engagement Platform | 4h | Select the sales engagement platform that will orchestrate multi-channel sequences. End state: Sales engagement platform selected and procurement approved.<br /><br />• Document requirements: sequence types, channel support, CRM integration, reporting needs<br />• Evaluate options: Outreach, Salesloft, Groove, AmpleMarket based on client CRM and budget<br />• Assess AI capabilities (email writing assistance, send time optimization, reply classification)<br />• Review sequence builder flexibility (branching logic, A/B testing, task creation)<br />• Compare admin controls and compliance features (sending limits, opt-out management)<br />• Finalize selection and initiate procurement process |
| 5. Evaluate Workflow Automation Tools | 3h | Select tools for building automated data flows and list generation workflows. End state: Workflow automation tool selected (e.g., Clay) if needed.<br /><br />• Assess need for advanced workflow automation beyond sales engagement platform<br />• Evaluate Clay, Make, or native CRM automation capabilities<br />• Determine use cases: waterfall enrichment, lead scoring, automatic list updates<br />• Compare build-vs-buy effort for complex automation requirements<br />• Document integration architecture between all selected tools<br />• Finalize tool stack and create procurement timeline |

### Task List: (Automated Outbound) 2. Data Infrastructure & Sequences
**Contains:** Parts 3-4

| Task | Est | Description |
|------|-----|-------------|
| 6. Define ICP and Build Account Lists | 5h | Create the target account list based on ICP criteria using data enrichment tools. End state: Clean, enriched target account list loaded into CRM.<br /><br />• Translate ICP criteria into data filters (industry, employee count, tech stack, geography)<br />• Pull initial target account list from data provider (2,000-10,000 accounts typical)<br />• Deduplicate against existing CRM accounts to avoid overlap with current customers/prospects<br />• Enrich account records with firmographic and technographic data<br />• Score accounts based on fit criteria and prioritize into tiers (A/B/C)<br />• Load account list into CRM with proper tagging and list assignment |
| 7. Build Contact Lists with Multi-Source Enrichment | 6h | Identify and enrich key contacts within target accounts using a waterfall enrichment approach. End state: Contact database with verified emails and complete profiles.<br /><br />• Define target personas per account (titles, departments, seniority levels)<br />• Pull contacts from primary data source (ZoomInfo, Apollo, etc.)<br />• Run waterfall enrichment to fill gaps (secondary sources for missing emails/phones)<br />• Verify email addresses using dedicated verification tool (NeverBounce, ZeroBounce)<br />• Enrich contacts with personalization data (LinkedIn URL, recent company news, tech stack)<br />• Load contacts into CRM linked to accounts with proper ownership assignment |
| 8. Configure Data Sync and Maintenance Workflows | 5h | Set up automated data flows to keep target lists fresh and synchronized. End state: Automated data pipelines running between enrichment tools, CRM, and sales engagement platform.<br /><br />• Configure bi-directional sync between CRM and sales engagement platform<br />• Set up automated enrichment triggers for new accounts/contacts added to CRM<br />• Create workflows to flag bounced emails and update contact status<br />• Build automated list refresh cadence (weekly/monthly re-enrichment)<br />• Configure duplicate detection and merge rules<br />• Document data flow architecture and troubleshooting procedures |
| 9. Design Multi-Channel Sequence Strategy | 4h | Create the overall sequence architecture including cadence timing, channel mix, and branching logic. End state: Sequence design document with mapped touchpoints and timing.<br /><br />• Define sequence types needed (cold outreach, warm follow-up, re-engagement, event-triggered)<br />• Map touchpoint cadence: typically 8-12 touches over 14-21 days<br />• Determine channel distribution (e.g., 60% email, 25% phone, 15% LinkedIn)<br />• Design branching logic based on prospect engagement (opens, clicks, replies)<br />• Create exit criteria and handoff rules to AE team<br />• Document A/B testing strategy for messaging and timing optimization |
| 10. Build Email Templates with Personalization | 5h | Create email templates with personalization tokens that scale relevance without sacrificing efficiency. End state: Library of tested email templates ready for sequence deployment.<br /><br />• Write email templates for each stage of sequence (opener, value prop, social proof, break-up)<br />• Build personalization framework using available data points (company, role, tech stack, pain point)<br />• Create industry/vertical-specific variations for top segments<br />• Configure personalization tokens in sales engagement platform<br />• Set up fallback values for missing personalization data<br />• Test templates with sample sends to internal team before launch |
| 11. Build Call Scripts and LinkedIn Touchpoints | 4h | Create phone scripts and LinkedIn messaging templates for non-email touchpoints. End state: Complete multi-channel playbook with scripts for every touchpoint.<br /><br />• Write phone scripts with discovery questions and objection handling<br />• Create voicemail scripts (keep under 30 seconds with clear CTA)<br />• Build LinkedIn connection request templates (personalized, under character limit)<br />• Create LinkedIn message templates for follow-up after connection<br />• Document call task disposition options and required logging fields<br />• Build quick-reference guide for SDRs with channel-specific best practices |
| 12. Configure Sequences in Sales Engagement Platform | 6h | Build the sequences in the selected platform with all touchpoints, timing, and automation rules. End state: Sequences built, tested, and ready for pilot launch.<br /><br />• Create sequence shells with proper naming conventions and tagging<br />• Add steps with correct timing and channel assignments<br />• Configure automation rules (task creation, send time optimization, reply detection)<br />• Set up sequence assignment rules and enrollment criteria<br />• Configure reply handling and sentiment classification<br />• Test sequences end-to-end with internal "dummy" contacts before live launch |

## Milestone 2: Automated Outbound Process - 2. Automation & Rollout
**Tag:** `automated-outbound`
**Description:** Implement automation and integrations, conduct pilot and full rollout, train team, and hand off
**Hours:** 48h

### Task List: (Automated Outbound) 3. Automation & Rollout
**Contains:** Parts 5-6

| Task | Est | Description |
|------|-----|-------------|
| 13. Configure Sequence Enrollment Automation | 5h | Set up rules for automatically enrolling prospects into sequences based on triggers and criteria. End state: Automated enrollment workflows running without manual intervention.<br /><br />• Define enrollment triggers (list membership, lead score threshold, intent signal)<br />• Build enrollment workflows in CRM or automation tool<br />• Configure mutual exclusion rules to prevent sequence overlap<br />• Set up daily enrollment caps to protect sender reputation<br />• Create re-enrollment rules for prospects who completed sequences without converting<br />• Test automation flows with sample records before enabling live enrollment |
| 14. Set Up Task and Follow-Up Automation | 5h | Configure automated task creation for manual touchpoints and follow-up actions. End state: SDRs receive automatically prioritized task queues each day.<br /><br />• Configure call task creation with priority and due date logic<br />• Set up task routing based on territory or account ownership<br />• Create automated follow-up tasks when prospects engage (open, click, reply)<br />• Configure meeting booked notifications and AE handoff tasks<br />• Set up overdue task escalation alerts<br />• Build dashboard showing task completion rates and aging |
| 15. Configure CRM Integration and Activity Logging | 5h | Ensure all outbound activities sync correctly to CRM for reporting and visibility. End state: Complete activity history visible in CRM with proper attribution.<br /><br />• Map sales engagement platform activities to CRM activity types<br />• Configure email, call, and LinkedIn activity logging<br />• Set up meeting booked sync with opportunity creation rules<br />• Configure engagement scoring to update CRM lead/contact scores<br />• Build campaign attribution for outbound touches<br />• Test activity sync and verify data appears correctly in CRM reports |
| 16. Conduct Pilot Launch with Select SDRs | 8h | Launch sequences with a small group of SDRs to validate performance before full rollout. End state: Pilot complete with validated performance benchmarks.<br /><br />• Select 2-3 SDRs for pilot program with clear success criteria<br />• Enable sequences for pilot group with monitored enrollment<br />• Track key metrics daily during pilot (deliverability, opens, replies, meetings)<br />• Gather SDR feedback on workflow and template effectiveness<br />• Identify and fix issues before full rollout<br />• Document pilot learnings and optimization recommendations |
| 17. Train Sales Development Team | 6h | Enable the full SDR team on new tools, processes, and sequences. End state: Team fully trained and confidently executing new outbound process.<br /><br />• Schedule training sessions (60-90 minutes recommended)<br />• Cover: tool navigation, sequence enrollment, task management, reply handling<br />• Walk through personalization best practices and when to customize<br />• Train on CRM logging requirements and data hygiene expectations<br />• Create quick-reference guides and video recordings for ongoing reference<br />• Set up office hours for first two weeks post-launch for questions |
| 18. Launch Full Rollout and Monitor Performance | 10h | Enable automated outbound for full team and establish ongoing performance monitoring. End state: Full team live with dashboards tracking key metrics.<br /><br />• Enable sequences and automation for full SDR team<br />• Configure real-time dashboards tracking sequence metrics (enrollment, replies, meetings)<br />• Set up weekly review cadence to analyze performance<br />• Create alerts for anomalies (deliverability drops, reply rate changes)<br />• Document escalation path for technical issues<br />• Establish SLA for issue resolution with tool vendors |
| 19. Hand Off and Enable Continuous Optimization | 9h | Transfer ownership to client team with documentation and optimization playbook. End state: Client self-sufficient with clear process for ongoing optimization.<br /><br />• Transfer admin credentials and access to client RevOps team<br />• Deliver documentation package (tool configurations, workflow diagrams, troubleshooting guides)<br />• Create optimization playbook with A/B testing framework<br />• Train client on how to analyze data and make sequence adjustments<br />• Schedule 30-day and 60-day check-in calls<br />• Close out project and transition to ongoing support if applicable |
